# AWS認定　セキュリティ・専門知識

## 概要

- AWS認定　セキュリティ専門知識の模擬試験の内容を復習しています
- 問題について、正しい選択はどれですか



## 問題
セキュリティエンジニアは、Amazon EC2 インスタンスで発生する可能性のあるセキュリティイベントの調査と対応 のためのプロセスを開発する必要があります。

すべてのEC2インスタンスは Amazon Elastic Block Store (Amazon EBS) を利用しており、AWS Systems Managerを使って管理されています。すべてのEC2インスタンスには Systems Manager Agent (SSM Agent) がインストールされています。

開発するプロセスは AWSのセキュリティベストプラクティスに準拠 し、以下の要件を満たす必要があります。

揮発性メモリ（RAM）および不揮発性メモリ（EBS）を保持 し、フォレンジック調査を行うこと

EC2インスタンスのメタデータを更新 し、該当のインシデントチケット情報を記録すること

インスタンスは調査中もオンラインのままとしつつ、マルウェアの拡散を防ぐため隔離 すること

揮発性データの収集中に発生するすべての活動を記録 すること

これらの要件を、最小限の運用負担 で満たすために適切な手順はどれですか？（3つ選択）

## 選択1
Systems Manager State Manager を使用してEBSボリュームのスナップショットを作成 し、インスタンスに適切なタグを追加する。

## 選択2
Systems Manager Run Command を使用 し、スクリプトを実行して揮発性データを収集する。

## 選択3
EBSボリュームのスナップショットを作成 し、インシデントチケット情報などのメタデータをタグとして追加 する。

## 選択4
関連するメタデータを収集し、終了保護を有効化 する。

インスタンスのセキュリティグループを更新し、アクセスを制限する。

Auto Scaling グループからインスタンスをデタッチ し、Elastic Load Balancer (ELB) から登録解除 する。

## 選択5
SSH または RDP を使用してEC2インスタンスに接続 し、スクリプトを実行して揮発性データを収集する。

## 選択6
関連するメタデータを収集し、終了保護を有効化 する。

隔離専用のサブネットに移動し、すべての送受信トラフィックを拒否 する。

Auto Scaling グループからインスタンスをデタッチ し、Elastic Load Balancer (ELB) から登録解除 する。
