- 問題について、正しい選択はどれですか

## 問題

グローバルなヘルスケアプロバイダーが最近、専門的な医学研究機関と合併し、その能力を拡大しました。最高情報セキュリティ責任者（CISO）として、あなたはAWS Organizationsを使用して両組織の複数のAWSアカウントを統合する任務を負っています。個々のアカウントと組織単位（OU）のアクセス権限を管理するプロセスを効率化するために、必要なサービスコントロールポリシー（SCP）を準備しました。しかし、1つのアカウントが医学研究データを保存するための新しいS3バケットを作成しようとした際に問題が発生しています。この問題の根本原因を調査する必要があります。そのアカウントには以下のSCPが適用されています：

{

  "Version": "2012-10-17",

  "Statement": [

    {

      "Effect": "Allow",

      "Action": "cloudtrail:*",

      "Resource": "*"

    },

    {

      "Effect": "Allow",

      "Action": "iam:*",

      "Resource": "*"

    }

  ]

}



アカウントの各IAMユーザーには以下のIAMポリシーが適用されています：



{

  "Version": "2012-10-17",

  "Statement": [

    {

      "Effect": "Allow",

      "Action": "s3:*",

      "Resource": [

        "arn:aws:s3:::*"

      ]

    },

    {

      "Effect": "Deny",

      "NotAction": "s3:*",

      "NotResource": [

        "arn:aws:s3:::*"

      ]

    }

  ]

}



この問題の最も可能性の高い原因は何ですか？

## 選択1
IAMポリシーにはユーザーがS3関連の操作を実行するために必要な権限が不足しています。

## 選択2

IAMポリシーとSCPの両方が問題の原因です。SCPはS3バケットの作成を明示的に許可し、IAMポリシーはSCPで指定された権限を正確に反映する必要があります。

## 選択3
SCPはAWSリソースのアクションをアローリストに登録することをサポートしていません。

## 選択4
SCPがアカウントがS3バケットを作成するために必要なアクションを明示的に許可していません。
